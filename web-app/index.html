Why it exists

Give AI agents Shelby access without giving them the entire machine.

AI agents need a safe bridge into Shelby storage. Raw filesystem access is too wide, and one-off wrappers are usually missing policy, sandboxing, and discoverable MCP UX.

`shelby-mcp` provides a local-first MCP layer that keeps file access scoped, validates uploads, exposes structured tools/resources/prompts, and stays usable in mock mode long before live integration is required.

What it solves

  • Safe and structured Shelby workflows for MCP hosts
  • Strict root scoping and safe-path narrowing for local file operations
  • Mock provider for rapid local testing and CI
  • Clear migration path toward deeper Shelby-native integration

Quick start

Three commands to a local mock-backed server.

shell
npm install
npm run setup
npm run dev:mock

What happens next

  • `setup` creates `.env` only if it is missing.
  • `dev:mock` prepares a safe workdir and forces mock provider mode.
  • Build with `npm run build` before pointing Cursor or VS Code at the compiled server.
  • Use mock mode first, then move to the experimental real provider path.

MCP client setup

Built for local STDIO integration in real editor workflows.

Cursor

Local STDIO

Build the server, register the compiled entrypoint, and start in mock mode first. Keep logs on `stderr`, not `stdout`, so MCP traffic stays clean.

.cursor/mcp.json
{
  "mcpServers": {
    "shelby": {
      "type": "stdio",
      "command": "node",
      "args": ["/absolute/path/to/shelby-mcp/dist/apps/server-stdio/src/index.js"],
      "env": {
        "SHELBY_PROVIDER": "mock",
        "SHELBY_WORKDIR": "/absolute/path/to/project/.shelby-workdir",
        "ALLOW_DESTRUCTIVE_TOOLS": "false"
      }
    }
  }
}

VS Code

`mcp.json`

Point `.vscode/mcp.json` at the built server and keep the sandbox root inside the workspace folder for predictable local behavior.

.vscode/mcp.json
{
  "servers": {
    "shelby": {
      "type": "stdio",
      "command": "node",
      "args": ["${workspaceFolder}/dist/apps/server-stdio/src/index.js"],
      "env": {
        "SHELBY_PROVIDER": "mock",
        "SHELBY_WORKDIR": "${workspaceFolder}/.shelby-workdir",
        "ALLOW_DESTRUCTIVE_TOOLS": "false"
      }
    }
  }
}
